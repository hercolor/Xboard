# PRD — Xboard Phase 3 仅后台模式 Web 入口收口

## 1. 背景

Xboard 已完成后台专属认证 API 与后台前端认证切换。当前后端项目内仍保留用户 Web 前台入口 `/`，会渲染主题壳层。这与“仅后台模式”目标冲突，尤其在前端已独立分离后，后端不再需要继续承担用户面板壳层渲染职责。

## 2. 目标

将后端项目的根路径 `/` 从“用户前台主题入口”收口为“后台入口跳转点”，并继续保留：

- `/{secure_path}` 后台 SPA
- `/api/v1/*`
- `/api/v2/*`
- `/s/{token}` 订阅分发
- 其他基础设施 API / 回调通道

## 3. 非目标

- 不删除任何 API 路由
- 不删除共享认证接口
- 不修改后台前端逻辑
- 不修改订阅分发、支付回调、节点、Webhook
- 不处理 Phase 4 的 API 退役

## 4. 用户与场景

### 4.1 目标用户
- 后台管理员
- 已分离前端系统的调用方
- 节点/客户端/回调等非浏览器通道

### 4.2 关键场景
1. 访问后端站点根路径 `/` 时，不再进入用户前台主题，而是转入后台路径 `/{secure_path}`
2. 已分离前端继续通过 `/api/*` 访问后端
3. 订阅链接 `/s/{token}` 继续正常分发

## 5. 设计原则

### 5.1 路由原则
- 只改 Web 根路径 `/`
- 保持后台路径 `/{secure_path}` 不变
- 保持订阅路径与 API 路径不变

### 5.2 迁移原则
- 采用低风险后台跳转，而不是直接 404
- 移除对主题壳层初始化的运行依赖，避免后端项目继续维护用户主题生命周期

## 6. 功能需求

### 6.1 根路径收口
- 原行为：渲染 `theme::<theme>.dashboard`
- 新行为：重定向到 `/{secure_path}`

### 6.2 后台入口保留
- `/{secure_path}` 继续渲染 `resources/views/admin.blade.php`
- 不改其参数与加载方式

### 6.3 API 与订阅保留
- `/api/v1/*` 与 `/api/v2/*` 不作改动
- `/s/{token}` 不作改动

## 7. 工程边界

- `routes/web.php` 应去除仅服务 `/` 主题渲染所需的无关依赖引用
- 不再从 `/` 路由触发 `ThemeService`、`File::copyDirectory`、主题初始化和视图渲染逻辑

## 8. 验收标准

1. `/` 不再渲染用户主题壳层
2. `/` 会跳转到 `/{secure_path}`
3. `/{secure_path}` 路由保持原样
4. `/s/{token}` 路由保持原样
5. API 层无改动
6. `routes/web.php` 通过 `php -l`

## 9. 实施顺序

1. 补齐 Ralph context / PRD / Test Spec
2. 改造 `routes/web.php` 根路径逻辑
3. 更新任务清单 Phase 3 状态
4. 静态验证与架构复核
5. 提交
